Section 1

System Overview

LEMS is the Techbridge University College (TUC) platform for collecting and analysing student evaluations of lecturers. It is designed to meet GTEC accreditation requirements for systematic evaluation of teaching quality across all academic programmes.

The system enables students to submit confidential, anonymous evaluations via their TUC institutional email identity, while administrators review aggregate results, manage the curriculum catalogue, and export data for academic board review.

Key Principles

🔒

Anonymous by Design

Student identities are never stored. A salted SHA-256 hash deduplicates submissions without recording who submitted.

🎓

GTEC Aligned

Curriculum data — programmes, courses, and credits — mirrors the GTEC-accredited structure for TUC Design programmes.

🔗

WMS SSO

All authentication routes through the WMS JWT system. Students and admins log in with their @techbridge.edu.gh accounts.

📊

Live Analytics

Administrators see rating distributions, per-lecturer averages, and programme-level trends in real time.

Evaluation Criteria

Each evaluation rates the lecturer on 10 criteria, each scored on a 1–5 scale (1 = Strongly Disagree, 5 = Strongly Agree).

Deduplication

How anonymity is enforced Each submission generates a salted SHA-256 hash of studentEmail + lecturerId + courseId using a server-side secret (LEMS_DEDUPE_SALT). The hash is stored; the email is never stored. A student who submits twice for the same lecturer+course combination receives a 409 Conflict — one vote per student per pairing, identity-free.

Recommend Field

Students also answer a single recommendation question: "Would you recommend this lecturer to other students?" with values YES, NO, or NEUTRAL.

Curriculum import (PDF → AI extraction → additive upsert), audit log viewer and system health status.

🧪

Self Test

Automated connectivity and API health checks — verifies all endpoints are reachable before evaluation periods.

Curriculum Import

Administrators can upload a programme PDF from the Admin Panel. The system passes the text to the WMS Gemini relay, which extracts lecturer names and courses. The admin reviews and confirms before an additive upsert is applied — existing evaluations are never touched.

Import safety The curriculum import is strictly additive. Lecturers are matched by full name; courses by name within the programme. Nothing is ever deleted. Evaluations are completely isolated from curriculum mutations.

Audit Log

Every admin action (programme created/updated/deleted, lecturer created/updated/deleted, course mutation, curriculum import) is written to wms_lems_audit_log with a timestamp, event type, status and the acting user's identity.

Section 7

Student Portal

The Student Portal is the public-facing evaluation form, accessible at lems.techbridge.edu.gh. Any user who can authenticate via WMS SSO with a @techbridge.edu.gh email may submit evaluations.

Before You Begin Checklist

Before submitting, students are shown a checklist reminding them that evaluations are anonymous, that they can only submit once per lecturer+course combination, and that honest feedback helps improve teaching quality.

Steps to Submit

  1. Visit lems.techbridge.edu.gh and log in with your TUC email.
  2. Select your Programme from the dropdown.
  3. Select the Course you are evaluating.
  4. The Lecturer assigned to the course is shown — confirm it is the correct person.
  5. Rate the lecturer on each of the 10 criteria (1–5 scale).
  6. Answer the recommend question (Yes / Neutral / No).
  7. Optionally add a written comment.
  8. Click Submit Evaluation. A confirmation screen confirms receipt.
One submission per pairing Attempting to evaluate the same lecturer+course combination a second time returns a 409 error. Your first submission is preserved and your identity is never revealed.

Section 9

Security

Authentication

LEMS delegates all authentication to the WMS SSO system. There are no local passwords. Login is via @techbridge.edu.gh email + WMS credentials. The access token has a 15-minute TTL; the refresh token lasts 7 days and rides in an HttpOnly cookie.

Authorisation Roles

WMS RoleLEMS Access LevelPermitted Actions
SYSTEM_ADMINFull AdminAll read and write operations including programme/lecturer/course mutations and audit log.
HODFull AdminSame as SYSTEM_ADMIN.
ADMIN_STAFFFull AdminSame as SYSTEM_ADMIN.
Any @techbridge.edu.ghStudentSubmit evaluations, view own submissions. Cannot access admin dashboard.

Student Anonymity

dedupeHash = SHA-256( studentEmail + "|" + lecturerId + "|" + courseId + LEMS_DEDUPE_SALT )

// Stored: dedupeHash  (64-char hex)
// Never stored: studentEmail, student name, student ID
Salt management The LEMS_DEDUPE_SALT is a 64-char hex secret stored only in /opt/tuc-wms/.env (chmod 600). If the salt is changed, all existing deduplication hashes become invalid — students can re-submit. Do not rotate the salt without an intentional evaluation reset.
Section 10

Deployment

Frontend Deployment

Run from the lems/ directory on the developer's machine:

.\deploy.ps1 -Build

This script:

  1. Pushes the current commit to GitHub
  2. SSH-connects to root@66.226.72.199
  3. Sparse-clones the lems/ subfolder from GitHub
  4. Runs pnpm install && VITE_API_URL=https://wms.techbridge.edu.gh/api/lems pnpm build
  5. Rsyncs dist/ to /var/www/vhosts/techbridge.edu.gh/lems-redirect/
  6. Fixes permissions and writes the SPA .htaccess

Backend (WMS) Deployment

LEMS runs inside the WMS JAR — there is no separate backend to deploy for LEMS alone. Deploying the WMS also deploys LEMS. The WMS systemd service is tuc-wms.service on the production server.

scp target/app.jar root@66.226.72.199:/opt/tuc-wms/app.jar
ssh root@66.226.72.199 "systemctl restart tuc-wms"

Database Migrations

Schema and data changes must be applied manually. The WMS uses ddl-auto: update which adds new columns automatically on restart, but does not seed data.

scp migration.sql root@66.226.72.199:/tmp/
ssh root@66.226.72.199 "PASS=$(cat /etc/psa/.psa.shadow) && mysql -u admin -p$PASS tuc_wms_db < /tmp/migration.sql"
