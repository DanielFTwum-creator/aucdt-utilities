# AGENT.md — Bridge Radio

Quick orientation for an AI agent working in this app. Authoritative environment
spec is **CONSTRAINTS.md** (read it first).

## What this is

A public HLS internet radio player (React 19 + hls.js) for TUC's `afrobeats` /
`music` / `neosoul` streams, plus an optional AI "find the lyrics" feature. No
login — anyone can listen. Served at the Plesk root vhost `radio.techbridge.edu.gh/`,
not under `ai-tools.techbridge.edu.gh/<slug>/`.

## Shape

- `src/App.tsx` — the whole player UI (single large component).
- `src/main.tsx`, `src/index.css`, `src/types.ts`.
- `server.ts` — Express: `/api/proxy` (CORS/M3U8-rewrite proxy for the HLS host),
  `/api/lyrics` (WMS-relayed Gemini lyrics lookup), `/api/health`; serves `dist/`
  under `NODE_ENV=production`, Vite dev middleware otherwise.
- HLS streams themselves are static files on `ai.techbridge.edu.gh`, generated by
  a separate ffmpeg script (`ai.techbridge.edu.gh/scripts/convert_to_hls.sh`) —
  not part of this repo.

## Rules that bite (see CONSTRAINTS.md + PATTERNS.md)

- **Never hold a Gemini key client-side** — `/api/lyrics` relays to WMS with
  `GEMINI_PROXY_KEY` (Pattern 11); no `@google/genai` import in `src/`, no `define`
  for `GEMINI_API_KEY` in `vite.config.ts`.
- **Root-vhost routing lives in `vhost_ssl.conf`, not `.htaccess`** — Plesk overrides
  `.htaccess` on root vhosts (Pattern 15). Canonical copy: `docs/vhost_ssl.conf`.
- `server.ts` runs under PM2 via `tsx` (Pattern 13) — keep `tsx` in `dependencies`.
- `server.ts`'s hardcoded port fallback (3021) is stale — the real port is 3032,
  supplied by `deploy.ps1` as an explicit env var. Don't "fix" the fallback to match
  SERVER_PORTS.md without checking `deploy.ps1` still overrides it.
- No auth layer exists in this app — don't assume `useAuth()` or a token wrapper
  is available when adding features.

## Build / run

- `pnpm install` → `pnpm build` (Vite) → `pnpm dev` (tsx server + Vite middleware)
  or `pnpm start` (production mode, serves `dist/`).
- `pnpm lint` runs `tsc --noEmit`.
- Deploy: `cd bridge-radio` then `.\deploy.ps1 -Build` (server-side build, rsync
  `dist/`, writes `vhost_ssl.conf`, PM2 start/reload, end-to-end MIME check).
