## 2. Architecture (how it works)

- **Player** — React + **hls.js**, plays static HLS from the stream host.
- **Streams** — static HLS on `https://ai.techbridge.edu.gh/<genre>/master.m3u8`
  (`afrobeats`, `music`, `neosoul`), generated by `ai.techbridge.edu.gh/scripts/convert_to_hls.sh`
  (ffmpeg). To add songs: drop `.m4a` in the genre dir, run the script.
- **Backend** (`server.ts`) provides two routes:
  - `GET /api/proxy?url=` — CORS proxy / m3u8 URL rewriter for the HLS host.
  - `GET /api/lyrics?track=&genre=` — server-side AI lyrics (key never hits the client).

## 3. Hard Constraints

1. **NEVER expose the Gemini key to the client.** No `@google/genai` import in `src/`,
   no `define` for `GEMINI_API_KEY` in `vite.config.ts`. Lyrics go through
   `/api/lyrics`, which resolves the key via the WMS proxy (root `PATTERNS.md` #11).
2. **Root vhost routing lives in `vhost_ssl.conf`, NOT `.htaccess`** — Plesk overrides
   `.htaccess` on root vhosts. SPA fallback MUST use `%{DOCUMENT_ROOT}%{REQUEST_URI}`
   (not `%{REQUEST_FILENAME}`). Canonical copy: [`docs/vhost_ssl.conf`](./docs/vhost_ssl.conf).
   See `PATTERNS.md` #15 (this is what caused the rms outage).
3. **`server.ts` is run by PM2 via `tsx`** (Pattern 13) — `tsx` is a runtime dependency,
   not devDependency. It listens on `0.0.0.0:$PORT` so Apache proxies `127.0.0.1:3032`.
4. **Package manager: pnpm.** Node v26 on the server (`nvm use 26`).

## 5. Deploy

```powershell
cd bridge-radio
.\deploy.ps1 -Build      # server-side build + rsync dist/, deploy server.ts,
                         # pnpm --prod, write vhost_ssl.conf, pm2 start/reload,
                         # then end-to-end MIME check over HTTPS
```

The radio was historically deployed **static-only**; this script moves it to the
standard node-backed model (required for `/api/lyrics`). Deploying changes the live
vhost from static to proxied — confirm before first run.
