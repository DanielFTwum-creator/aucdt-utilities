# PlayGrow — Smart Fun for Bright Minds

## Purpose

An AI-for-Good educational game platform for young learners at Techbridge University College. Children explore a World Map of learning zones and play fully interactive mini-games that make artificial intelligence tangible through direct, playful experience. Every game is framed around a real AI use-case (pattern recognition, image labelling, language generation, spatial reasoning) so kids leave with intuition about what AI actually does.

## Stack

- **Framework:** React 19 + TypeScript
- **Build tool:** Vite 7
- **Styling:** Tailwind CSS 4.x with scoped `.playgrow-shell` CSS custom property tokens
- **Deployment:** Docker — `node:24-alpine` builder → `nginx:alpine` server
- **Package manager:** pnpm

## Key Decisions

**No backend.** All game logic is client-side. The admin password is embedded in source (acceptable for an internal institutional tool; this is not a public SaaS product and the admin panel contains only mock actions).

**Pointer Events API for drag-and-drop.** PuzzleBuilder and BuildItBlocks use `element.setPointerCapture(pointerId)` so pointer move and up events fire on the originating element even when the pointer leaves it. Ghost position is updated imperatively via a `ref.current.style.transform` — zero React re-renders per pointer-move frame, works on both mouse and touch.

**Scoped `.playgrow-shell` design tokens.** `index.css` sets `--pg-bg`, `--pg-card`, `--pg-accent`, etc. under `.playgrow-shell`. Tailwind components reference them via arbitrary `bg-[var(--pg-accent)]` syntax. This keeps the PlayGrow palette isolated from other apps in the same monorepo build pipeline.

**`freshDeal()` over lazy `useMemo`.** StoryMaker originally used `useMemo([], [])` for card pools — they never refreshed while the component was mounted. Moved to `useState` + a `freshDeal()` builder so "New" can call it at any time. Same pattern applied to BuildItBlocks (`challengeIdx` state) and PaintWorld (`challengeIdx` state).

**Auto-reveal on mount.** StoryMaker deals and auto-reveals a complete story on first render so children land on content, not a blank picker. All other games start their experience automatically (PatternPath begins its sequence on mount, FindMatch builds a deck on mount, etc.).

**Deep story library.** StoryMaker draws from 30 WHO × 30 DID × 30 WHERE entries spanning world literature — Anansi, King Arthur, Florence Nightingale, Sherlock Holmes, Cleopatra, Mansa Musa, Odysseus, and more — yielding 27,000 unique sentence combinations. Each story is wrapped with random OPENERS and CLOSERS arrays for narrative framing.

**CSS keyframe classes over inline animation styles.** Animations (`pg-pop`, `pg-bubble-in`) are declared in `index.css` and applied via class names to satisfy the inline-style linter rule.

## Open Questions

- PaintWorld canvases are ephemeral (in-memory `<canvas>` only). If save/share is added, `canvas.toBlob()` → download link is the minimal path.
- Language, Movement, Social, Exploration, and Rest zones show text stubs generated by an AI activity service. These are candidates for the next round of interactive game development.
- The admin panel controls (Reset Progress, Push Update, Backup, Flush Cache) are mock actions that log to an audit panel. Real backend integration would require a TUC Auth API and a server-side action endpoint.
