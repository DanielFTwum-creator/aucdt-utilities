\section{Introduction}

\subsection{Purpose}
This document defines the functional and non-functional requirements for the DocuLaTeX application. Its primary purpose is to provide a clear and unambiguous specification for a web-based tool that converts documents from common formats (PDF, Markdown, HTML) into a themed LaTeX output.

\subsection{Document Conventions}
This document follows the IEEE 830-1998 standard. All requirements are uniquely identified for traceability. British International English spelling and grammar are used throughout.

\subsection{Intended Audience and Reading Suggestions}
This document is intended for project managers, web developers, UI/UX designers, the quality assurance team, and project stakeholders involved in the development of DocuLaTeX.

\subsection{Product Scope}
\begin{cardcontainer}[title={Product Scope}]
    \textbf{DocuLaTeX} is a web-based service designed to automate the process of formatting documents into a high-quality, consistent, and professional layout using a predefined LaTeX theme. It accepts user-uploaded PDF, Markdown, and HTML files and generates a downloadable PDF and the source \texttt{.tex} file as output.

    \vspace{1em}
    \poppinssemibold\color{AUCDTPrimaryText}Version 1.0 (MVP) Scope:\normalfont The initial release will focus on the core conversion engine delivered through a simple web interface. The scope includes: a user account system, a file upload mechanism, a server-side parsing module for each supported format, a mapping engine to convert parsed elements to the specified LaTeX theme, and the generation of the final PDF and \texttt{.tex} files for download.
\end{cardcontainer}

\subsection{References}
\begin{itemize}[label={\color{AUCDTBurgundy}\faBook}]
    \item IEEE Std 830-1998, \textit{Recommended Practice for Software Requirements Specifications}.
    \item AUCDT Brand Identity Guide v2.0, August 2025.
    \item The LaTeX source of \textit{srs.tex} for CounselorPro, which serves as the target theme definition.
\end{itemize}

\section{Overall Description}

\subsection{Product Perspective}
DocuLaTeX is a new, self-contained, standalone web application. It will be accessible via a web browser and will not require any local software installation for the end-user. The server-side component will manage all processing, including LaTeX compilation.

\subsection{Product Functions}
The major functions are: User Account Management, File Upload, Content Parsing, LaTeX Generation and Compilation, and Output File Delivery (Preview and Download).

\subsection{User Classes and Characteristics}
\begin{cardcontainer}[title={User Classes}]
\begin{description}[style=unboxed]
    \item[\poppinssemibold Academics \& Researchers:] Need to quickly format papers and articles into a consistent style without installing or managing a LaTeX environment.
    \item[\poppinssemibold Technical Writers:] Require a tool to automate the conversion of documentation from Markdown or HTML into print-ready PDFs.
    \item[\poppinssemibold Students:] Want an easy, accessible way to make their reports and assignments look professional.
\end{description}
\end{cardcontainer}

\subsection{Operating Environment}
The application will be a web-based platform hosted on a secure cloud infrastructure. It must be accessible through modern web browsers (latest versions of Chrome, Firefox, Safari, Edge) on desktop devices. The server environment must include a full LuaLaTeX distribution for document compilation.

\subsection{Design and Implementation Constraints}
\begin{notebox}
    The primary constraint is the fidelity of the conversion. The generated PDF must be a high-fidelity representation of the source content, styled according to the target theme. The server must securely handle user-uploaded files.
\end{notebox}
\begin{itemize}[label={\color{AUCDTBurgundy}\faExclamationTriangle}]
    \item \textbf{C-1 (Server-Side LaTeX):} The server environment is responsible for all LaTeX compilation. This requires robust sandboxing to securely process user-provided content.
    \item \textbf{C-2 (Parsing Limitations):} The parsing engines must be robust but are not expected to handle poorly-formed or exceptionally complex source files in V1.0. For PDFs, only text-based documents are supported.
    \item \textbf{C-3 (Theme Hardcoding):} The target LaTeX theme (based on the AUCDT style) is hardcoded for V1.0.
    \item \textbf{C-4 (Browser Compatibility):} The user interface must be functional and visually consistent on the latest two versions of major desktop web browsers.
\end{itemize}

\subsection{Assumptions and Dependencies}
\begin{description}[style=unboxed]
    \item[\poppinssemibold Assumption:] Users have a stable internet connection for uploading and downloading files.
    \item[\poppinssemibold Assumption:] Input files are well-structured. For example, HTML uses semantic tags, and PDFs have selectable text.
    \item[\poppinssemibold Dependency:] The system depends on third-party cloud hosting and server-side libraries for parsing and file management.
\end{description}

\section{Specific Requirements}

\subsection{External Interface Requirements}
\subsubsection{User Interfaces (UI)}
The primary interface will be a graphical web interface, designed to be clean, intuitive, and conformant to the AUCDT brand identity.
\begin{cardcontainer}[title={Web UI Specifications}]
    The web interface shall consist of the following key components:
    \begin{itemize}[label={\color{AUCDTGold}\faDesktop}]
        \item \textbf{Dashboard:} A main page displaying user's recent documents and an option to start a new conversion.
        \item \textbf{File Upload Zone:} A clearly marked area where users can drag-and-drop or browse to select a single file (.pdf, .md, .html) for conversion.
        \item \textbf{Processing View:} An interface that shows the status of the conversion (e.g., "Uploading," "Parsing," "Compiling LaTeX," "Done").
        % CORRECTED: Escaped the ampersand character to prevent a compilation error.
        \item \textbf{Preview \& Download Panel:} Once processing is complete, this panel will show a preview of the generated PDF and provide clearly labelled buttons to download the final PDF and the source \texttt{.tex} file.
    \end{itemize}
\end{cardcontainer}

\subsection{Functional Requirements}
\begin{longtable}{p{0.15\textwidth} p{0.75\textwidth}}
    \hline
    \rowcolor{AUCDTBurgundy!80}
    \poppinssemibold\color{AUCDTWhite} ID & \poppinssemibold\color{AUCDTWhite} Requirement Description \\
    \hline
    \endhead
    
    \multicolumn{2}{l}{\bfseries\color{AUCDTPrimaryText} FR-1: User Account Management} \\
    \textbf{FR-1.1} & The system shall allow users to register for a new account using an email and password. \\
    \textbf{FR-1.2} & The system shall allow registered users to log in and log out. \\
    \textbf{FR-1.3} & The system shall associate uploaded documents and their outputs with the logged-in user's account. \\

    \multicolumn{2}{l}{\bfseries\color{AUCDTPrimaryText} FR-2: File Upload and Validation} \\
    \textbf{FR-2.1} & The system shall provide an interface for an authenticated user to upload a single file. \\
    \textbf{FR-2.2} & The system shall validate that the uploaded file has a supported extension (.pdf, .md, .html) and is below a maximum size of 10 MB. \\
    
    \multicolumn{2}{l}{\bfseries\color{AUCDTPrimaryText} FR-3: Content Processing} \\
    \textbf{FR-3.1} & The system shall extract structural elements from the uploaded file (headers, lists, tables, etc.). \\
    \textbf{FR-3.2} & The system shall map parsed elements to corresponding LaTeX commands based on the predefined theme. \\
    \textbf{FR-3.3} & The system shall compile the generated \texttt{.tex} file on the server to produce a PDF. \\

    \multicolumn{2}{l}{\bfseries\color{AUCDTPrimaryText} FR-4: Output Delivery} \\
    \textbf{FR-4.1} & The system shall display an embedded preview of the first page of the generated PDF. \\
    \textbf{FR-4.2} & The system shall provide a button for the user to download the full generated PDF file. \\
    \textbf{FR-4.3} & The system shall provide a button for the user to download the generated source \texttt{.tex} file. \\
    \hline
\end{longtable}

\subsection{Non-Functional Requirements}
\subsubsection{Performance}
% CORRECTED: Changed the list label syntax to a more robust method.
\begin{itemize}
    \item[{\color{AUCDTCampusGreen}\faTachometerAlt}] \textbf{NFR-1.1:} Web pages shall load in under 3 seconds on a standard broadband connection.
    \item[{\color{AUCDTCampusGreen}\faTachometerAlt}] \textbf{NFR-1.2:} The end-to-end conversion process for a 10-page document shall be completed within 30 seconds.
    \item[{\color{AUCDTCampusGreen}\faTachometerAlt}] \textbf{NFR-1.3:} The system shall support up to 20 concurrent conversion processes.
\end{itemize}

\subsubsection{Security}
\begin{itemize}
    \item[{\color{AUCDTCampusGreen}\faShieldAlt}] \textbf{NFR-2.1:} All user passwords must be hashed using a strong, modern algorithm (e.g., Argon2).
    \item[{\color{AUCDTCampusGreen}\faShieldAlt}] \textbf{NFR-2.2:} User-uploaded content must be handled in a secure, sandboxed environment to prevent execution of malicious code on the server.
\end{itemize}

\subsubsection{Software Quality Attributes}
\begin{itemize}
    \item[{\color{AUCDTCampusGreen}\faStar}] \textbf{NFR-3.1 (Correctness):} The generated PDF must be a high-fidelity, visually correct representation of the source document formatted with the theme.
    \item[{\color{AUCDTCampusGreen}\faStar}] \textbf{NFR-3.2 (Usability):} The web interface must be intuitive, requiring no training for a user to upload a file and get a result.
\end{itemize}

\subsection{Error Handling and Recovery}
\begin{itemize}
    \item[{\color{AUCDTBurgundy}\faExclamationCircle}] \textbf{EH-1:} If a user uploads an unsupported file type or a file that exceeds the size limit, the UI shall display a clear, user-friendly error message in real-time.
    \item[{\color{AUCDTBurgundy}\faExclamationCircle}] \textbf{EH-2:} If the server-side compilation fails, the system shall inform the user with a message like, "We couldn't convert your document. Please check if the file is corrupted or badly formatted and try again."
\end{itemize}
