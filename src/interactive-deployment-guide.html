0 Project Setup: Create Your React App

Before you deploy, you'll need a React project. If you haven't already, you can create a new React application using `pnpm create react-app` (or `npx create-react-app` if you prefer npm). Replace `my-react-app` with your desired project name.

pnpm create react-app my-react-app

Once created, navigate into your new project directory:

cd my-react-app

1 First: Build Your Application

Before you can deploy your site, you need to create a production-ready build. This process compiles and optimizes your code into a set of static files. Navigate to your project's root directory in your terminal and run the following command.

pnpm build

This will create a `build` or `dist` folder in your project. This is the folder you will deploy.

2 Choose Your Platform

Select a platform below to view the specific deployment instructions.

Deploying with Netlify

Step 1: Create Account & Connect Git. Sign up for a free account at netlify.com and connect it to your GitHub, GitLab, or Bitbucket account.
Step 2: Import Project. From your dashboard, click "Add new site" > "Import an existing project". Select your Git provider and choose the repository for your project.
Step 3: Configure Build Settings. Netlify usually auto-detects Create React App settings. Confirm they match the following:
  • Branch to deploy: `main` (or your primary branch)
  • Build command: `pnpm build`
  • Publish directory: `build`
Step 4: Deploy. Click "Deploy site". Netlify will build and deploy your application. Any future pushes to the connected branch will automatically trigger a new deployment.

Deploying with Vercel

Step 1: Create Account & Import Project. Sign up for a free account at vercel.com. From the dashboard, click "New Project" and import your desired Git repository.
Step 2: Configure Project. Vercel will auto-detect that you are using a React application and configure the settings for you. Verify them:
  • Framework Preset: `Create React App`
  • Build Command: `pnpm build`
  • Output Directory: `build`
Step 3: Deploy. Click "Deploy". Vercel will handle the rest. Continuous deployment is automatically enabled for your production branch.

Deploying with GitHub Pages

Step 1: Install `gh-pages` package. This helper package simplifies deploying to GitHub Pages.
pnpm install --save-dev gh-pages
Step 2: Update `package.json`. You need to add two new entries to your `package.json` file.

Add a `homepage` property at the top level:

"homepage": "https://<username>.github.io/<repo-name>",

Add `predeploy` and `deploy` scripts inside the `scripts` object:

"scripts": { "predeploy": "pnpm build", "deploy": "gh-pages -d build" }

Remember to replace `<username>` and `<repo-name>` accordingly.

Step 3: Deploy. Run the deploy script from your terminal.
pnpm deploy
Step 4: Configure Repository Settings. In your GitHub repository settings, go to the "Pages" section. Ensure the source is set to deploy from the `gh-pages` branch. Your site will be live after a few minutes.

General Tips

Environment Variables

For sensitive data like API keys, use your hosting provider's environment variable settings instead of hardcoding them in your app. This keeps your secrets safe.

Custom Domains

All of these platforms allow you to connect a custom domain. Check their documentation for specific instructions on how to point your domain to your new site.

Troubleshooting

If a deployment fails, the first place to look is the build logs on your hosting platform. They provide detailed error messages that can help you diagnose the issue.
